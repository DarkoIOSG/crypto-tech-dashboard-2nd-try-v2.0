# 任务交接指南：加密货币技术指标 Dashboard 开发

## 一、项目概述（给开发者看的简要说明）

我需要你帮我开发一个 **加密货币技术指标分析 Web Dashboard**。这是一个部署在 Mac 服务器上的全栈应用，用来查看 CoinGecko 市值前 200 个加密货币的技术面强度、趋势/反转评分和排名。

**核心功能：**
- 获取 Top 200 代币的日线 OHLCV 数据（通过多个交易所 API）
- 计算 12 个技术指标家族（SMA 交叉、MACD、RSI、布林带、KDJ 等）
- 给出趋势强度和反转强度两个综合评分 + 百分位排名
- 深色主题 K 线图 + 各指标图表（类 TradingView 风格）
- 每日自动更新 + 手动刷新

**技术栈：**
- 后端：Python FastAPI + pandas + CCXT（多交易所数据获取）
- 前端：原生 JS + TradingView Lightweight Charts v4（开源）
- 数据存储：本地 CSV 文件缓存
- 定时任务：APScheduler
- 无需 Node.js / Webpack 构建

## 五、开发中的关键注意事项

### 最容易出错的地方

1. **RSI 必须用 Wilder 平滑**：`ewm(alpha=1/period, adjust=False)`，不是 SMA。如果用错了，RSI 值会和 TradingView 差很多
2. **MACD 要归一化到价格**：`(ema_fast - ema_slow) / Close`，不是直接相减
3. **CCXT 符号格式统一用 `BTC/USDT`**（带斜杠），不是 `BTCUSDT`
4. **CSV 文件用 CoinGecko ID 命名**（如 `bitcoin.csv`），不是交易所符号
5. **图表同步是强需求**：主 K 线图缩放时，所有子图表必须联动。用 master-slave 模式 + 全局锁防止事件循环
6. **首次启动会很慢**（40-70 分钟），需要显示进度条
7. **原子写入**：CSV append 必须用 `.tmp` + `os.rename()` 策略

### 代码质量要求

- 所有配置常量集中在 `config.py`，不要硬编码
- API Key 放 `.env` 文件，不要写在代码里
- 错误处理：API 调用要有重试（指数退避），网络异常不能让整个更新流程崩溃
- 日志：关键操作（数据拉取、指标计算、评分更新）要有日志输出

```bash

# 2. 创建虚拟环境
```
